What is a Pattern?

A pattern is a reusable solution that can be applied to commonly occurring problems in software design - in our case - in writing JavaScript web applications.

Design patterns have three main benefits:

  • Patterns are proven solutions
  • Patterns can be easily reused
  • Patterns can be expressive

Categories of Design Patterns

Creational Design Patterns

Creational design patterns focus on handling object creation mechanisms where objects are created in a manner suitable for the situation we're working in.

  • Abstract factory
  • Builder
  • Factory method
  • Prototype
  • Singleton

Structural Design Patterns

Structural patterns are concerned with object composition and typically identify simple ways to realize relationships between different objects.

  • Adapter
  • Bridge
  • Composite
  • Decorator
  • Facade
  • Flyweight
  • Proxy

Behavioral Design Patterns

Behavioral patterns focus on improving or streamlining the communication between disparate objects in a system.

  • Command
  • Interpreter
  • Iterator
  • Mediator
  • Memento
  • Observer
  • State
  • Strategy

Factory Pattern

Factory Pattern is class-based creational pattern. In this, we provide a generic interface that delegates the responsibility of object instantiation to its subclasses.

Singleton Pattern

Singleton is a special creational design pattern where only one instance of a class can exist.

Flyweight Pattern

This is a structural design pattern focused on efficient data sharing through fine-grained objects. It is used for efficiency and memory conservation purposes.

Chain of Responsibility Pattern

This is a behavioural design pattern that provides a chain of loosely coupled objects. Each of these objects can choose to act on or handle the request of the client.

Conclusion

Design patterns are crucial to software engineering and can be very helpful in solving common problems.
